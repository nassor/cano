Scheduler

Automate your workflows with flexible scheduling and concurrency.

The Scheduler provides workflow scheduling capabilities for background jobs and automated workflows. It supports intervals, cron expressions, and manual triggers.

Scheduling Strategies

Interval

Run workflows at fixed time intervals.

scheduler.every_seconds(...)

Cron

Run workflows based on cron expressions.

scheduler.cron(..., "0 0 9 * * *")

Manual

Trigger workflows on-demand via API.

scheduler.manual(...)

Scheduling Strategy Examples

The Scheduler supports multiple scheduling strategies. Here are complete examples for each.

1. Interval Scheduling - Fixed Time Intervals

Run workflows at regular time intervals. Best for periodic tasks like health checks or data syncing.

gantt title Interval Scheduling (Every 30 seconds) dateFormat ss axisFormat %Ss section Workflow Run 1 :0, 2s Wait :2, 28s Run 2 :30, 2s Wait :32, 28s Run 3 :60, 2s 
use cano::prelude::*;
use async_trait::async_trait;

#[derive(Debug, Clone, PartialEq, Eq, Hash)]
enum State { Start, Complete }

#[derive(Clone)]
struct HealthCheckTask;

#[async_trait]
impl Task for HealthCheckTask {
    async fn run(&self, store: &MemoryStore) -> Result, CanoError> {
        println!("Running health check...");
        
        // Check system health
        let status = "healthy".to_string();
        store.put("last_health_check", status)?;
        
        Ok(TaskResult::Single(State::Complete))
    }
}

#[tokio::main]
async fn main() -> Result<(), CanoError> {
    let mut scheduler = Scheduler::new();
    let store = MemoryStore::new();

    let workflow = Workflow::new(store.clone())
        .register(State::Start, HealthCheckTask)
        .add_exit_state(State::Complete);

    // Run every 30 seconds
    scheduler.every_seconds("health_check", workflow, State::Start, 30)?;

    scheduler.start().await?;
    Ok(())
}

2. Cron Scheduling - Time-Based Expressions

Run workflows based on cron expressions. Perfect for scheduled reports, backups, or time-specific tasks.

gantt title Cron Scheduling (Daily at 9 AM and 6 PM) dateFormat HH axisFormat %H:00 section Workflow Run 1 :09, 1h Run 2 :18, 1h %% Add empty space to ensure full visibility Space :20, 0h 
use cano::prelude::*;
use async_trait::async_trait;
use chrono::Utc;

#[derive(Clone)]
struct DailyReportNode {
    report_type: String,
}

#[async_trait]
impl Node for DailyReportNode {
    type PrepResult = Vec;
    type ExecResult = String;

    async fn prep(&self, store: &MemoryStore) -> Result {
        println!("📊 Preparing {} report...", self.report_type);
        
        // Load data for report
        let data = vec!["metric1".to_string(), "metric2".to_string(), "metric3".to_string()];
        store.put("report_start", Utc::now().to_rfc3339())?;
        
        Ok(data)
    }

    async fn exec(&self, data: Self::PrepResult) -> Self::ExecResult {
        println!("📊 Generating report with {} records", data.len());
        
        // Generate report
        format!("{} report: {} records processed", self.report_type, data.len())
    }

    async fn post(&self, store: &MemoryStore, result: Self::ExecResult) -> Result {
        println!("📊 Report completed: {}", result);
        store.put("last_report", result)?;
        
        Ok(State::Complete)
    }
}

#[tokio::main]
async fn main() -> Result<(), CanoError> {
    let mut scheduler = Scheduler::new();
    let store = MemoryStore::new();

    // Morning report workflow
    let morning_report = Workflow::new(store.clone())
        .register(State::Start, DailyReportNode { 
            report_type: "Morning".to_string() 
        })
        .add_exit_state(State::Complete);

    // Evening report workflow
    let evening_report = Workflow::new(store.clone())
        .register(State::Start, DailyReportNode { 
            report_type: "Evening".to_string() 
        })
        .add_exit_state(State::Complete);

    // Run daily at 9 AM: "0 0 9 * * *"
    scheduler.cron("morning_report", morning_report, State::Start, "0 0 9 * * *")?;
    
    // Run daily at 6 PM: "0 0 18 * * *"
    scheduler.cron("evening_report", evening_report, State::Start, "0 0 18 * * *")?;

    scheduler.start().await?;
    Ok(())
}

3. Manual Triggering - On-Demand Execution

Trigger workflows manually via API. Ideal for user-initiated tasks or event-driven processing.

sequenceDiagram participant API as API Request participant S as Scheduler participant W as Workflow API->>S: trigger("data_export") S->>W: Start Workflow W-->>S: Complete S-->>API: Success 
use cano::prelude::*;

#[derive(Clone)]
struct DataExportTask;

#[async_trait]
impl Task for DataExportTask {
    async fn run(&self, store: &MemoryStore) -> Result, CanoError> {
        println!("Starting data export...");
        
        // Export data to CSV
        let export_path = "/tmp/export.csv".to_string();
        store.put("export_path", export_path)?;
        
        println!("Export completed");
        Ok(TaskResult::Single(State::Complete))
    }
}

#[tokio::main]
async fn main() -> Result<(), CanoError> {
    let mut scheduler = Scheduler::new();
    let store = MemoryStore::new();

    let export_workflow = Workflow::new(store.clone())
        .register(State::Start, DataExportTask)
        .add_exit_state(State::Complete);

    // Register as manual-only workflow
    scheduler.manual("data_export", export_workflow, State::Start)?;

    // Start scheduler in background
    let scheduler_handle = scheduler.clone();
    tokio::spawn(async move {
        scheduler_handle.start().await.unwrap();
    });

    // Trigger manually when needed
    println!("Triggering export...");
    scheduler.trigger("data_export").await?;
    
    // Can be triggered again later
    tokio::time::sleep(Duration::from_secs(5)).await;
    scheduler.trigger("data_export").await?;

    scheduler.stop().await?;
    Ok(())
}

4. Mixed Scheduling - Combining Strategies

Use multiple scheduling strategies together for complex automation scenarios.

gantt title Mixed Scheduling Strategies dateFormat HH:mm axisFormat %H:%M section Interval Tasks Sync Every 5min :00:00, 24h section Cron Tasks Daily Backup :03:00, 1h Weekly Report :09:00, 1h section Manual Tasks Emergency Export :done, 14:30, 15m 
use cano::prelude::*;

#[tokio::main]
async fn main() -> Result<(), CanoError> {
    let mut scheduler = Scheduler::new();
    let store = MemoryStore::new();

    // Define simple tasks
    #[derive(Clone)]
    struct DataSyncTask;

    #[async_trait]
    impl Task for DataSyncTask {
        async fn run(&self, _store: &MemoryStore) -> Result, CanoError> {
            println!("Syncing data...");
            Ok(TaskResult::Single(State::Complete))
        }
    }
    
    #[derive(Clone)]
    struct BackupTask;
    #[async_trait]
    impl Task for BackupTask {
        async fn run(&self, _store: &MemoryStore) -> Result, CanoError> {
            println!("Running backup...");
            Ok(TaskResult::Single(State::Complete))
        }
    }
    
    #[derive(Clone)]
    struct WeeklyReportTask;

    #[async_trait]
    impl Task for WeeklyReportTask {
        async fn run(&self, _store: &MemoryStore) -> Result, CanoError> {
            println!("Generating weekly report...");
            Ok(TaskResult::Single(State::Complete))
        }
    }
    
    #[derive(Clone)]
    struct EmergencyExportTask;
    
    #[async_trait]
    impl Task for EmergencyExportTask {
        async fn run(&self, _store: &MemoryStore) -> Result, CanoError> {
            println!("Emergency export...");
            Ok(TaskResult::Single(State::Complete))
        }
    }

    // 1. Interval: Data sync every 5 minutes
    let sync_workflow = Workflow::new(store.clone())
        .register(State::Start, DataSyncTask)
        .add_exit_state(State::Complete);
    
    scheduler.every_seconds("data_sync", sync_workflow, State::Start, 300)?;

    // 2. Cron: Daily backup at 3 AM
    let backup_workflow = Workflow::new(store.clone())
        .register(State::Start, BackupTask)
        .add_exit_state(State::Complete);
    
    scheduler.cron("daily_backup", backup_workflow, State::Start, "0 0 3 * * *")?;

    // 3. Cron: Weekly report on Mondays at 9 AM
    let report_workflow = Workflow::new(store.clone())
        .register(State::Start, WeeklyReportTask)
        .add_exit_state(State::Complete);
    
    scheduler.cron("weekly_report", report_workflow, State::Start, "0 0 9 * * MON")?;

    // 4. Manual: Emergency data export
    let export_workflow = Workflow::new(store.clone())
        .register(State::Start, EmergencyExportTask)
        .add_exit_state(State::Complete);
    
    scheduler.manual("emergency_export", export_workflow, State::Start)?;

    // Start scheduler
    let scheduler_handle = scheduler.clone();
    tokio::spawn(async move {
        scheduler_handle.start().await.unwrap();
    });

    // Monitor and trigger as needed
    loop {
        tokio::time::sleep(Duration::from_secs(60)).await;
        
        // Check status of all workflows
        let workflows = scheduler.list().await;
        for info in workflows {
            println!("{}: {:?} (runs: {})", info.id, info.status, info.run_count);
        }
        
        // Example: Trigger emergency export if needed based on some condition
        // scheduler.trigger("emergency_export").await?;
    }
}

Graceful Shutdown

The scheduler supports graceful shutdown, allowing currently running workflows to complete before stopping. 

// Stop scheduler but allow running flows to finish
scheduler.stop().await?;

Advanced Pattern: Multi-Level Map-Reduce

Combine manual workflow triggering with split/join to create powerful multi-level map-reduce patterns. Each workflow processes a batch of data in parallel (workflow-level map-reduce), and multiple workflows run concurrently with different parameters (scheduler-level map-reduce).

Complete Example: Multi-Region Data Processing

This example demonstrates processing data from multiple regions concurrently. Each region's workflow processes its records in parallel, then all results are aggregated globally. 

use cano::prelude::*;
use async_trait::async_trait;
use std::sync::Arc;
use tokio::sync::RwLock;
use tokio::time::Duration;

#[derive(Debug, Clone, PartialEq, Eq, Hash)]
enum ProcessingState {
    Start,
    ProcessRecords,
    Summarize,
    Complete,
}

// Shared global state for collecting results from all regions
#[derive(Clone)]
struct GlobalResults {
    summaries: Arc>>,
}

impl GlobalResults {
    fn new() -> Self {
        Self {
            summaries: Arc::new(RwLock::new(Vec::new())),
        }
    }
    
    async fn add_summary(&self, summary: RegionSummary) {
        self.summaries.write().await.push(summary);
    }
    
    async fn get_all_summaries(&self) -> Vec {
        self.summaries.read().await.clone()
    }
}

#[derive(Debug, Clone)]
struct RegionSummary {
    region: String,
    total_records: usize,
    total_value: f64,
    avg_value: f64,
}

// Task to initialize region processing
#[derive(Clone)]
struct InitRegionTask {
    region: String,
    record_ids: Vec,
}

#[async_trait]
impl Task for InitRegionTask {
    async fn run(&self, store: &MemoryStore) -> Result, CanoError> {
        println!("🌍 [{}] Initializing with {} records", self.region, self.record_ids.len());
        
        store.put("region", self.region.clone())?;
        store.put("record_ids", self.record_ids.clone())?;
        
        Ok(TaskResult::Single(ProcessingState::ProcessRecords))
    }
}

// Task to process a single record (used in split)
#[derive(Clone)]
struct ProcessRecordTask {
    region: String,
    record_id: u32,
}

#[async_trait]
impl Task for ProcessRecordTask {
    async fn run(&self, store: &MemoryStore) -> Result, CanoError> {
        println!("  📊 [{}] Processing record {}", self.region, self.record_id);
        
        // Simulate I/O operation (database query, API call, etc.)
        tokio::time::sleep(Duration::from_millis(100)).await;
        
        // Generate mock value
        let value = (self.record_id as f64) * 10.5;
        
        // Store result
        store.put(&format!("record_{}", self.record_id), value)?;
        
        println!("  ✅ [{}] Record {} processed: ${:.2}", self.region, self.record_id, value);
        
        Ok(TaskResult::Single(ProcessingState::Summarize))
    }
}

// Task to summarize region results
#[derive(Clone)]
struct SummarizeRegionTask {
    global_results: GlobalResults,
}

#[async_trait]
impl Task for SummarizeRegionTask {
    async fn run(&self, store: &MemoryStore) -> Result, CanoError> {
        let region: String = store.get("region")?;
        let record_ids: Vec = store.get("record_ids")?;
        
        println!("  📈 [{}] Summarizing results...", region);
        
        // Collect all processed values
        let mut total_value = 0.0;
        for record_id in &record_ids {
            if let Ok(value) = store.get::(&format!("record_{}", record_id)) {
                total_value += value;
            }
        }
        
        let summary = RegionSummary {
            region: region.clone(),
            total_records: record_ids.len(),
            total_value,
            avg_value: total_value / record_ids.len() as f64,
        };
        
        println!(
            "  ✅ [{}] Complete: {} records, total: ${:.2}, avg: ${:.2}",
            region, summary.total_records, summary.total_value, summary.avg_value
        );
        
        // Add to global results
        self.global_results.add_summary(summary).await;
        
        Ok(TaskResult::Single(ProcessingState::Complete))
    }
}

// Create workflow for a specific region
fn create_region_workflow(
    region: String,
    record_ids: Vec,
    global_results: GlobalResults,
) -> Workflow {
    let store = MemoryStore::new();
    
    Workflow::new(store)
        .register(
            ProcessingState::Start,
            InitRegionTask {
                region: region.clone(),
                record_ids: record_ids.clone(),
            },
        )
        // Split: Process all records in parallel
        .register_split(
            ProcessingState::ProcessRecords,
            record_ids
                .iter()
                .map(|&id| ProcessRecordTask {
                    region: region.clone(),
                    record_id: id,
                })
                .collect::>(),
            JoinConfig::new(
                JoinStrategy::All,
                ProcessingState::Summarize,
            )
            .with_timeout(Duration::from_secs(30)),
        )
        .register(
            ProcessingState::Summarize,
            SummarizeRegionTask {
                global_results: global_results.clone(),
            },
        )
        .add_exit_state(ProcessingState::Complete)
}

// Global reduce function
async fn reduce_global_results(global_results: &GlobalResults) {
    println!("\n🌐 GLOBAL REDUCE: Aggregating all regions");
    println!("==========================================");
    
    let summaries = global_results.get_all_summaries().await;
    
    if summaries.is_empty() {
        println!("⚠️  No regions completed");
        return;
    }
    
    // Display region summaries
    println!("\n📦 Region Summaries:");
    for summary in &summaries {
        println!("  • {}: {} records, total: ${:.2}, avg: ${:.2}",
            summary.region,
            summary.total_records,
            summary.total_value,
            summary.avg_value
        );
    }
    
    // Global statistics
    let total_records: usize = summaries.iter().map(|s| s.total_records).sum();
    let total_value: f64 = summaries.iter().map(|s| s.total_value).sum();
    let global_avg = total_value / total_records as f64;
    
    println!("\n📈 Global Statistics:");
    println!("  Total regions: {}", summaries.len());
    println!("  Total records: {}", total_records);
    println!("  Total value: ${:.2}", total_value);
    println!("  Global average: ${:.2}", global_avg);
}

#[tokio::main]
async fn main() -> Result<(), CanoError> {
    println!("🚀 Multi-Level Map-Reduce Example");
    println!("==================================\n");
    
    let mut scheduler = Scheduler::new();
    let global_results = GlobalResults::new();
    
    // Define regions with their record IDs (different parameters per workflow)
    let regions = vec![
        ("US-East".to_string(), vec![1, 2, 3]),
        ("EU-West".to_string(), vec![4, 5, 6]),
        ("Asia-Pacific".to_string(), vec![7, 8, 9]),
    ];
    
    // Register a workflow for each region
    for (region, record_ids) in ®ions {
        let workflow = create_region_workflow(
            region.clone(),
            record_ids.clone(),
            global_results.clone(),
        );
        
        scheduler.manual(region, workflow, ProcessingState::Start)?;
        println!("✅ Registered workflow: {} ({} records)", region, record_ids.len());
    }
    
    // Start scheduler
    let mut scheduler_clone = scheduler.clone();
    tokio::spawn(async move {
        scheduler_clone.start().await
    });
    
    tokio::time::sleep(Duration::from_millis(500)).await;
    
    // MAP PHASE: Trigger all region workflows
    println!("\n🗺️  MAP PHASE: Processing all regions...\n");
    for (region, _) in ®ions {
        scheduler.trigger(region).await?;
    }
    
    // Wait for completion
    println!("⏳ Waiting for all regions to complete...\n");
    for _ in 0..20 {
        tokio::time::sleep(Duration::from_secs(1)).await;
        
        let workflows = scheduler.list().await;
        let completed = workflows.iter().filter(|w| w.status == Status::Completed).count();
        
        if completed == workflows.len() {
            println!("✅ All {} regions completed!\n", completed);
            break;
        }
    }
    
    // REDUCE PHASE: Aggregate global results
    println!("🔄 REDUCE PHASE: Aggregating results\n");
    reduce_global_results(&global_results).await;
    
    scheduler.stop().await?;
    println!("\n✅ Multi-level map-reduce complete!");
    
    Ok(())
}

Key Concepts

Level 1: Workflow Split/Join

Map: Each workflow splits its work into parallel tasks (e.g., 3 records processed simultaneously)

Reduce: Results join back together into a region summary

Level 2: Scheduler Map/Reduce

Map: Multiple workflows run concurrently, each with different parameters (different regions)

Reduce: After all workflows complete, aggregate results across all regions

Shared State

Use Arc<RwLock<>> to collect results from all workflows into a shared global state

Each workflow independently adds its summary to the global collection
